# 初始化新项目

这个 Skill 会帮用户创建一个新项目文件夹，并一次性配好项目记忆系统。除了四件套和 `README.md`，还必须创建 `memory_map.md`，让项目从第一天开始就具备场景路由能力。

## Claude 专用实现

本 Skill 为 Claude 与共享技能库准备，优先使用以下工具：

- `Glob`：检查目标目录与现有文件
- `Write`：创建新文件
- `Edit`：根据访谈结果补充内容
- `Read`：核对已创建内容
- `Bash`：仅用于创建目录

不使用其他 Bash 命令。

## 触发指令

- `/init-project <项目名称>`
- `/new <项目名称>`
- "新建项目"

## 执行逻辑

### 1. 确认位置

询问用户项目存放位置。若用户没有特别说明，默认使用当前目录。

### 2. 创建目录

创建项目根目录，并在根目录下创建 `AI专属文档/`。

### 3. 初始化记忆文件

创建以下文件：

- `AI专属文档/task_plan.md`
- `AI专属文档/findings.md`
- `AI专属文档/scratchpad.md`
- `AI专属文档/decisions.md`
- `AI专属文档/memory_map.md`
- `README.md`
- `AGENTS.md`

### 4. 文件模板

#### `AI专属文档/task_plan.md`

```markdown
# 任务计划

## 项目目标
- [ ] 待定义

## 当前阶段
- 待填写

## 下一步
- [ ] 补充目标、约束与验收标准
```

#### `AI专属文档/findings.md`

```markdown
# 发现与知识

## 已确认信息
- 待补充
```

#### `AI专属文档/decisions.md`

```markdown
# 决策复盘

## 记录模板

### 背景
- 待补充

### 决策
- 待补充

### 结果
- 待补充

### 反思
- 待补充
```

#### `AI专属文档/scratchpad.md`

```markdown
# 临时草稿

（临时草稿区）
```

#### `AI专属文档/memory_map.md`

```markdown
# 上下文地图

## 项目定位
- 一句话说明项目在做什么
- 当前主要产出类型

## 项目内记忆
- `task_plan.md`：当前目标、阶段与下一步
- `findings.md`：关键发现、已知约束、可复用知识
- `decisions.md`：关键取舍与历史判断
- `scratchpad.md`：尚未定稿的想法与草稿

## 外部数据源
- 名称：
  - 路径：
  - 用途：
  - 适用场景：
  - 优先级：高/中/低

## 场景路由
### 汇报与对齐口径
- 触发词：汇报、给领导看、复盘稿、口径、总结材料
- 优先读取：`findings.md` > 已确认数据源 > 历史汇报模板 > `decisions.md`
- 输出重点：已确认结论、关键数据、表达口径、风险点

### 培训方案与课程设计
- 触发词：培训方案、课程设计、课程大纲、项目方案、ADDIE、学习路径
- 优先读取：`findings.md` > 外部模板/案例库 > `decisions.md`
- 输出重点：目标对象、设计约束、已有模板、当前空缺

### 项目复盘与解释原因
- 触发词：复盘、回顾、为什么、当时怎么定的、取舍
- 优先读取：`decisions.md` > 最近 `memory/` 日志 > `findings.md`
- 输出重点：背景、决策、结果、反思

### 经验萃取与案例沉淀
- 触发词：萃取、案例、方法论、沉淀、经验库
- 优先读取：`findings.md` > `scratchpad.md` > 外部案例库
- 输出重点：可复用模式、适用边界、后续沉淀位置

### 历史追溯与上下文恢复
- 触发词：之前、历史、上次聊到哪、找资料、恢复上下文
- 优先读取：`memory_map.md` > `memory/` 最近日志 > `task_plan.md`
- 输出重点：最近讨论、未完成事项、相关数据源入口

### 执行推进
- 触发词：继续、推进、处理、跟进、补、改
- 优先读取：`task_plan.md` > `decisions.md` > `scratchpad.md`
- 输出重点：下一步、阻塞、最近决策
```

#### `README.md`

```markdown
# {项目名称}

## 简介

{项目简介}

## 当前主要产出

{例如：培训方案、课程大纲、汇报材料、复盘沉淀}

## 记忆入口

- 项目记忆目录：`AI专属文档/`
- 场景路由入口：`AI专属文档/memory_map.md`
```

#### `AGENTS.md`

```markdown
# 项目协作规则

## 记忆路由

- 遇到具体工作场景时，优先参考 `AI专属文档/memory_map.md`
- 读取项目记忆时，优先使用 `AI专属文档/`
- 保存项目记忆时，同步维护 `task_plan.md`、`findings.md`、`decisions.md`、`scratchpad.md`、`memory_map.md`
```

### 5. 需求访谈

创建完成后，至少补充以下两项信息：

- 项目的核心目标
- 当前主要产出类型

然后用这些信息更新：

- `task_plan.md` 的“项目目标”
- `README.md` 的“简介”和“当前主要产出”
- `memory_map.md` 的“项目定位”

## 完成提示

完成后应提示用户：项目记忆系统已就绪，`memory_map.md` 已作为场景路由入口创建完成，后续可直接用 `/init-memory` 进入项目。
