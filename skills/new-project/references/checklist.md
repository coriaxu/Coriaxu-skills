# New Project Checklist

## 1. 访谈规则

必须围绕以下五项收集信息：

- 项目做什么
- 为什么做
- 目标是什么
- 用户角色
- AI 角色

执行要求：

- 用户一次性说全时，直接提取，不要重复盘问
- 用户只说部分信息时，允许先创建
- 用户只敲 `/new-project` 时，必须先发起访谈
- 语气自然，像同事沟通，不要像表单问卷

## 2. 路径规则

- 用户明确给了路径，按用户路径创建
- 用户没给路径时，优先使用当前工作目录
- 当前工作目录不适合时，再补问一句项目要放在哪里

## 3. 文件映射规则

### `README.md`

- 项目名称：来自项目名
- 项目简介：由“做什么 + 为什么做”组成
- 目标：来自目标说明
- 负责人角色：来自用户角色
- AI 协作方式：来自 AI 角色
- 信息缺口：把尚未回答的字段写成“待补充”

### `AI_专属/task_plan.md`

- 项目目标：来自目标说明
- 当前已知情况：汇总五项访谈结果
- 当前待办：列出缺失项和首轮动作
- 下一步行动：根据已知信息给出最小下一步

### `AI_专属/findings.md`

- 已确认信息：汇总已知信息
- 待验证 / 待补充：列出未回答字段

### `AI_专属/decisions.md`

- 初始化时写明“当前暂无关键决策”
- 后续出现方案取舍、范围变化、关键判断时，从这里连续记录

### `AI_专属/scratchpad.md`

- 保留一版简洁访谈摘录
- 列出当前还缺哪些信息

## 4. 后续补录规则

当用户补充新信息时：

- 更新 `README.md`
- 更新 `AI_专属/task_plan.md`
- 更新 `AI_专属/findings.md`
- 更新 `AI_专属/scratchpad.md`
- 不清空 `AI_专属/decisions.md`

如果项目目录已经存在，应视为补录更新，不要重新创建一套空壳文件。
