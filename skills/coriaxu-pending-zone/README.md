# 【Coriaxu】Pending Zone

一个给个人使用的“悬置区” skill。

它适合保存这类事情：

- 现在觉得重要，但还没决定要不要做
- 需要以后再判断，不适合立刻放进待办
- 有相关材料和初步想法，但还没有明确结论

默认会在桌面创建：

```bash
~/Desktop/个人悬置区/
```

每个议题保存成一张 Markdown 卡片。当前还没结论的卡片放在根目录，已经有结论的卡片放进 `归档/`。

## 典型用法

```text
这个先悬置一下
```

创建一张新卡。

```text
看一下悬置区
```

列出当前还挂着的卡片。

```text
这个已经决定不做了，归档
```

更新状态并移入归档目录。

## 文件结构

```text
个人悬置区/
├── README.md
├── 2026-04-27-博客方案.md
└── 归档/
    └── 2026-04-20-旧工具方案.md
```

## 适合与不适合

适合放进悬置区的是“未决判断题”。例如：要不要尝试一个新工具、是否调整一个工作流程、某个想法是否值得以后展开。

不适合放进悬置区的是已经确定的执行任务、项目内待办、长期规则、单纯素材。
