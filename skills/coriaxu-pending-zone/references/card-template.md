# 悬置卡模板

每张卡是一个 Markdown 文件，文件名格式：

```text
YYYY-MM-DD-主题.md
```

主题用 6 到 18 个字描述具体议题。文件名不要包含空格、斜杠、冒号、引号。英文短语可用连字符连接。

## Frontmatter

只保留四个字段：

```yaml
---
created: 2026-04-27
status: 悬置中
source: agent
updated: 2026-04-27
---
```

字段说明：

- `created`：创建日期，用 `date +%Y-%m-%d` 获取
- `status`：`悬置中` / `已决定做` / `已决定不做`
- `source`：`agent` / `user`
- `updated`：最近更新时间

## 正文结构

```markdown
## 背景

这件事从哪里冒出来的，一两句话说明。

## 待判断的核心问题

- 现在还没有决定的点是什么
- 需要哪些条件才能做判断
- 如果之后要推进，第一步是什么

## 相关材料

- 链接、文件路径、截图路径或对话摘要

## 判断讨论记录

（初次建卡留空。后续回访时追加日期小节。）
```

## 归档区块

归档时在正文末尾追加：

```markdown
## 最终结论

### 2026-04-27

决定：做 / 不做

原因：一句话说明理由。
```

## 完整样例

```markdown
---
created: 2026-04-27
status: 悬置中
source: agent
updated: 2026-04-27
---

## 背景

看到一个新的博客框架，感觉可能适合个人写作，但暂时还没决定是否迁移。

## 待判断的核心问题

- 这套框架是否比现有写作方式更省心
- 是否接受一次性配置成本
- 现在就做，还是等后续统一整理写作系统

## 相关材料

- 项目链接：https://example.com
- 初步判断：功能完整，但配置成本需要再评估

## 判断讨论记录

（暂无）
```
