# 边界判断

悬置区只保存“未决判断题”。判断标准是：这件事已经具体到值得以后再看，但现在还没有决定要不要做、怎么做。

## 悬置区 vs 待办清单

进入待办清单的是已经决定要执行的事。

进入悬置区的是尚未决定是否执行的事。

示例：

- “周五前改完这篇文章” -> 待办
- “以后要不要把写作系统换成新工具” -> 悬置区

## 悬置区 vs 项目计划

项目计划绑定某个具体项目，记录已确定的任务和下一步。

悬置区跨项目，记录还没有形成任务的议题。

示例：

- “这个网页按钮要改成蓝色” -> 项目计划
- “以后要不要把这个网页做成一个长期产品” -> 悬置区

## 悬置区 vs 长期记忆

长期记忆保存已经确认的规则、偏好和背景。

悬置区保存还没确认的判断。

示例：

- “以后所有周报都只写进展和计划” -> 长期记忆
- “要不要新建一套周报自动化流程” -> 悬置区

## 悬置区 vs 素材库

素材库保存资料本身。

悬置区保存围绕资料产生的未决问题。

示例：

- 一篇文章链接 -> 素材库
- “这篇文章里的方法要不要改造成自己的流程” -> 悬置区

## 嗅探信号

“先放一放”“回头再说”“之后再想”只有在同时满足两个条件时才提醒：

- 当前对话里已经有一个具体议题
- 用户没有给出结论，只是暂时停止讨论

日常表达不触发，例如：

- “这个文件先放一放旁边”
- “这个话题先放一放，我们聊下一个”
- “我今天先放一放，明天再说”
