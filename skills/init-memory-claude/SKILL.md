# 初始化项目记忆

这个 Skill 是项目的“开机程序”。目标不是只确认文件是否存在，而是先判断当前请求是否真的需要项目记忆，再决定读取哪些资料、是否补齐记忆文件，以及现在最值得推进的下一步。

## Claude 专用实现

本 Skill 为 Claude 与 Gemini 这类文档工具环境准备，优先使用以下工具：

- `Glob`：查找目录、文件和候选路径
- `Read`：读取现有内容并提炼摘要
- `Write`：只在确实需要项目记忆时创建缺失文件
- `Edit`：补建模板或修正已有内容

除非宿主环境明确要求，否则不要依赖 Bash。

## 触发指令

- `/init-memory`
- `/memory`
- “初始化记忆”
- “读取项目进度”

## 核心职责

- 判断当前请求是否真的需要项目记忆
- 定位当前项目真正生效的记忆目录
- 仅在需要时补齐缺失的核心记忆文件与 `memory_map.md`
- 先读取项目级场景路由，再根据当前用户消息加载对应资料
- 输出一段可直接开工的中文启动简报

## 执行逻辑

### 1. 先判断当前请求是否需要项目记忆

先判断当前场景属于哪一类：

- 明确的项目恢复或项目推进
  - 用户就是要继续某个持续项目，或者当前目录已经明显是一个长期项目
  - 进入项目记忆读取流程
- 单一任务或只想看施工日志
  - 不自动创建项目记忆目录
  - 优先做轻量判断，必要时只补读施工日志

若当前消息不足以判断，再结合目录结构和已有文件决定；能不问就不问。

### 2. 定位真正生效的记忆目录

按以下优先级检查：

1. `AI专属文档/`
2. `AI_专属/`
3. 项目根目录中的旧版记忆文件

重点检查：

- `task_plan.md`
- `findings.md`
- `scratchpad.md`
- `decisions.md`
- `memory_map.md`

命中规则：

- 只要命中 `AI专属文档/` 或 `AI_专属/`，就优先使用专属目录
- 若只命中根目录旧文件，可继续兼容读取，但要提醒后续迁移
- 若没有命中任何稳定记忆目录，不要为了“存一下”自动制造项目记忆，除非当前任务明确需要长期项目管理

### 3. 决定读取顺序

默认顺序如下：

1. 先确认项目定义
   - `README.md`
2. 再确认项目规则
   - `CLAUDE.md`
   - `AGENTS.md`
3. 再读项目推进资料
   - `task_plan.md`
   - 最近一条有效 `decisions.md`
4. 只有在确实需要时，才继续读：
   - `memory_map.md`
   - `findings.md`
   - `scratchpad.md`
   - 最近施工日志

原则：

- 简单项目优先少读
- 规则文档只提炼约束，不把它们当项目简介
- 只有在当前问题真的依赖历史细节时，才展开更深一层读取

### 4. 补建规则

只有在已经确认“这是持续项目”之后，才允许补建缺失文件。

可补建：

- `decisions.md`
- `memory_map.md`

若完全没有项目记忆目录，但当前明确是长期项目，可在 `AI专属文档/` 下补建：

- `task_plan.md`
- `findings.md`
- `scratchpad.md`
- `decisions.md`
- `memory_map.md`

若当前更像单一任务或一次性处理：

- 不创建 `AI专属文档/`
- 不创建 `AI_专属/`
- 不创建记忆三件套

### 5. 输出要求

输出一段可直接开工的中文启动简报，至少包含：

- 当前命中的路径
- 当前项目是什么
- 命中的规则文档
- 当前目标
- 下一步
- 最近一次关键决策
- 当前约束或风险
- 是否补建了缺失文件

## 注意事项

1. 不要一上来就把所有记忆文件都读一遍。
2. 单一任务默认不创建项目记忆目录。
3. 优先读项目定义与规则，再决定要不要展开长期记忆。
4. 当用户只输入 `/init-memory` 时，不要反问一串开放问题，先给启动简报。
