# Save Memory Student

## 触发方式

- `/save-memory`
- `存一下`
- `记一下`
- `帮我记住`
- `存档`

## 适用边界

- 只回顾当前这一轮对话
- 只处理当前工作的存档，不自动存档
- 绝不改用户原有业务文件
- 不接入 Google Drive、CLAUDE.md、GEMINI.md 或其他全局规则文件
- AI 工作日报和项目记忆是两个独立写入目标

## 执行流程

1. 回顾当前对话，只提取事实性工作内容，不编造完成项、发现或决策。
2. 先判断有没有实质工作推进；如果没有，直接说明“没有需要存档的内容”，不写任何文件。
3. 生成一条准备写入桌面 `AI 工作日报` 的内容，至少包含项目标签、做了什么、发现了什么、下一步。
4. 只有在当前目录存在 `AI_专属/` 时，才同时准备项目记忆更新；不存在就安静跳过，不催用户先跑 `init-memory`。
5. 项目标签优先取当前工作目录名；目录名太泛时，再从对话内容推断；仍然不清楚时，写成 `[未分类]`。
6. 先把准备写入的内容展示给用户确认。用户没有明确同意前，不创建、不追加、不修改任何文件。
7. 用户确认后，写入 `~/Desktop/AI 工作日报/YYYY-MM-DD.md`。文件夹不存在就创建；同一天多次存档只追加，不覆盖。
8. 若同时更新项目记忆，只替换 `task_plan.md` 里的“当前阶段”“下一步”，并把发现、决策、临时想法追加到对应文件。

## 写入规则

- `AI 工作日报`：标题格式固定为 `## HH:MM - [项目名] 事项主题`
- `task_plan.md`：只更新最新状态快照
- `findings.md`：追加新发现，带日期
- `decisions.md`：追加新决策，且必须含“背景 + 决策 + 理由”
- `scratchpad.md`：追加未成熟想法，带日期

详细提取规则和写法见 `references/extract-rules.md`。触发校验样例见 `evals/trigger_cases.json`。

## 安全规则

- 学员版一律不使用云盘路径
- 没有 `AI_专属/` 时，只写桌面 `AI 工作日报`
- 只有用户主动触发时才执行，不自动弹出“要不要存一下”
- 面向用户的表述统一使用“AI 工作日报”“任务计划”“发现记录”“决策日志”“临时草稿”
