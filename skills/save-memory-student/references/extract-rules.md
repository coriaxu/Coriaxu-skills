# Save Memory Student 提取规则

## 1. AI 工作日报提取规则

只在当前对话里有实质工作时写入。可提取内容包括：

- 做了什么：完成的任务、整理的数据、写过的方案、讨论出的结论
- 发现了什么：新信息、异常点、意外发现、和预期不同的地方
- 下一步：还没做完但已经明确的下一步动作

如果只有问题讨论、没有实质推进，也可以记录成“讨论了什么”，但不要硬写成“已完成”。

## 2. 项目标签推断规则

优先顺序：

1. 当前工作目录名
2. 当前对话里反复出现的项目关键词
3. `[未分类]`

规则要点：

- 目录名已经很明确时，直接拿来用
- 目录名是“工作”“资料”“项目”这类泛词时，再从对话里提取更具体的项目名
- 推断不出来就标 `[未分类]`，不要为了标签反复追问用户

## 3. AI 工作日报写法

写入路径固定为：

- `~/Desktop/AI 工作日报/YYYY-MM-DD.md`

条目格式固定为：

```md
## HH:MM - [项目名] 事项主题
- 做了什么
- 发现了什么
- 下一步：……
```

同一天多次写入时，直接追加到同一个日期文件末尾。

## 4. 项目记忆更新规则

### 任务计划

只有出现这些情况才更新：

- 当前阶段推进了
- 原计划改变了
- 下一步已经更明确了

写法：

- `当前阶段` 和 `下一步` 用最新内容替换旧内容

### 发现记录

属于这些内容才追加：

- 新数据发现
- 方法优化
- 工具心得
- 行业或业务新洞察

写法：

```md
## YYYY-MM-DD
- …
```

### 决策日志

属于这些内容才追加：

- 明确做了取舍
- 缩小或扩大了范围
- 调整了优先级

每条都必须同时包含：

- 背景
- 决策
- 理由

示例：

```md
## 2026-04-10
### 背景
- 衍生品业务的数据口径不统一

### 决策
- 本期报告先不纳入衍生品对比

### 理由
- 现在强行对比会误导判断，先把现货业务结论做扎实
```

### 临时草稿

属于这些内容才追加：

- 还没验证的假设
- 还没拍板的方案方向
- 灵光一闪但来不及展开的思路

## 5. 确认展示格式

### 有 AI 工作日报 + 项目记忆

```md
我回顾了一下刚才的对话，有这些值得记住的：

**今天的 AI 工作日报**：
- 14:30 [项目名] …
- 下一步：…

**项目记忆更新**：
- 任务计划：…
- 新发现：…
- 决策记录：…

没问题的话我就写入。
```

### 只有 AI 工作日报

```md
我回顾了一下刚才的对话，有这些值得记住的：

**今天的 AI 工作日报**：
- 14:30 [项目名] …
- 下一步：…

没问题的话我就写入。
```

### 无需存档

```md
刚才这轮对话没有需要存档的内容，不用存。
```

## 6. 示例

### 示例一：单项目

```md
## 14:30 - [印度券商竞品调研] 佣金数据整理
- 完成了 Top 5 券商佣金数据整理
- 发现 Zerodha 3 月佣金结构有明显调整
- 下一步：周五前完成对比矩阵初稿
```

### 示例二：同一天两个项目

```md
## 10:20 - [Q2营销方案] 渠道优先级讨论
- 跟同事确认了三个主要投放渠道
- 发现抖音 CPL 比预估低 30%
- 下一步：明天跟供应商确认报价

## 16:40 - [月度经营分析] 数据交叉验证
- 完成了 3 月 GMV 数据校对
- 下一步：周四前输出分析初稿
```

### 示例三：目录不明确

```md
## 17:10 - [未分类] 周报素材梳理
- 讨论了本周最值得保留的三项进展
- 下一步：明早整理成汇报提纲
```
