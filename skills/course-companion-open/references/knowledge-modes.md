# Knowledge Modes

知识库连接是可降级模块。它的目标不是强迫用户拥有复杂笔记系统，而是在条件允许时，把新课程接到用户已有知识里。

## First-Use Question

当没有配置文件，也没有用户明确说明时，只问一次：

```text
这堂课要不要接入你已有笔记？选一个就行：不接入 / 扫描指定文件夹 / 接入 Obsidian 库。
```

用户回答后：

- 选择“不接入”：本次使用 `none`。
- 选择“扫描指定文件夹”：追问具体文件夹路径。
- 选择“接入 Obsidian 库”：追问 vault 根目录和要扫描的子目录。

不要在同一堂课里反复问配置。

## Mode: none

适合：

- 用户没有笔记库
- 用户暂时不想配置
- 当前只想围绕课程本身讨论

执行方式：

1. 不扫描旧文件。
2. 从课程骨架笔记中抽出 3-5 个内部锚点。
3. 每个锚点说明它为什么值得讨论。
4. 告诉用户：这些是课程内部锚点，不是旧笔记连接。

示例输出：

```md
这次不接入旧笔记。我先抓 3 个课程内部锚点：

1. **判断先于方法**：这堂课真正谈的不是技巧，而是一个人如何判断什么值得学。
2. **知识进入场景**：课程反复强调，知识只有接到真实任务里才会改变行为。
3. **反例比原则更有力量**：作者用失败案例说明边界，比直接给原则更有效。
```

## Mode: folder

适合：

- 用户有一批 Markdown 或文本笔记
- 用户没有使用 Obsidian
- 用户只想让助手扫描几个明确文件夹

执行方式：

1. 只扫描用户提供的路径。
2. 先匹配文件名、标题、frontmatter、tags。
3. 再读取少量最可能相关的文件。
4. 最多展示 3-5 个强相关连接。
5. 找不到时转入 `none` 的内部锚点模式。

不要做：

- 不要扫描用户没有授权的父目录。
- 不要为了凑数展示弱相关文件。
- 不要读取大型二进制文件，除非当前环境有可靠读取工具。

## Mode: obsidian

适合：

- 用户有 Obsidian vault
- 用户用 Markdown、wikilinks、tags 或 frontmatter 组织知识
- 用户希望新课程接入自己的长期笔记系统

执行方式：

1. 从 `vault_root` 进入。
2. 只扫描 `scan_paths` 中列出的子目录。
3. 优先匹配：
   - 文件名
   - 一级标题
   - frontmatter
   - tags
   - wikilinks
   - 与课程 Dots / Lines 高度相似的段落
4. 最多展示 3-5 个强相关连接。
5. 每个连接只解释一句，不长篇引用。

安全边界：

- 不自动扫描整个 vault。
- 不改写用户已有笔记，除非用户明确要求。
- 不新增知识卡，除非进入 Phase 3 且用户确认。

## Ranking Rule

选择强相关连接时，按这个顺序判断：

1. 同一概念或同一问题
2. 同一逻辑结构
3. 能补足课程盲区
4. 能挑战课程观点
5. 用户刚刚表达过关心

时间新旧只作为辅助，不作为主要排序标准。
