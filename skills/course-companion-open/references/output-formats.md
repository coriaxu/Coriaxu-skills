# Output Formats

Phase 3 在用户说“结束”“保存”“写成笔记”“出卡片”“finish”“save”等表达时触发。

## Final Study Note

默认文件名：

```text
YYYY-MM-DD-[课程名]-深度研读笔记.md
```

默认结构：

```md
---
type: course-study-note
course:
source:
date:
knowledge_mode: none | folder | obsidian
related_notes:
tags:
---

# [课程名] 深度研读笔记

## 1. 课程骨架

[放入 three-level-notes 的核心结构。可以压缩，但不要改变作者原意。]

## 2. 知识连接

### 2.1 已有笔记连接

- **[笔记标题或路径]**：一句话说明为什么相关。

### 2.2 课程内部锚点

- **[锚点]**：一句话说明为什么值得讨论。

> 如果 knowledge_mode 是 none，只保留“课程内部锚点”。

## 3. 对话增补

### 3.1 用户观点

> [!note] Learner
> 保留用户最关键的原话或改写后的观点。

### 3.2 Assistant 观点

> [!note] Companion
> 保留助手在对话中形成的关键判断。

### 3.3 追问与未解问题

- [问题 1]
- [问题 2]

## 4. 本课真正带走的东西

用 3-5 条写清楚这堂课给学习者带来的真实增量。

## 5. 可迁移场景

这堂课的观点可以迁移到哪些工作、学习或生活场景。

## 附录：Assistant 课后反应

如果 Phase 2 有完整的开场反应或阶段性判断，放在这里。
```

## Knowledge Card

只有满足任一条件时才生成：

- 用户明确要求单独写卡片
- 对话产生了一个独立概念
- 这个概念可以脱离课程复用
- 它能连接到用户已有笔记或未来任务

默认文件名：

```text
[概念名].md
```

默认结构：

```md
---
type: knowledge-card
source_course:
date:
tags:
---

# [概念名]

## 一句话

用一句话说清这个概念。

## 它从哪里来

说明它来自课程、对话、用户经验，还是三者的组合。

## 为什么重要

说明它解决了什么理解问题。

## 怎么用

列出 2-3 个可迁移场景。

## 相关连接

- [已有笔记或课程笔记]

## 原始片段

> [!quote] Learner / Companion / Course
> 保留关键原话，避免长篇堆叠。
```

## File Writing Rule

- 用户只说“给我看看”：只输出草稿。
- 用户说“保存”“写入”“生成文件”：按配置路径写文件。
- 没有输出路径时，先询问保存位置。
- 已有同名文件时，先询问覆盖、改名，或追加。
