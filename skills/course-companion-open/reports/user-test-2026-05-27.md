## First-Time User Simulation

User prompt:

```text
Use $course-companion-open to study this lesson with me.

这堂课讲的是：为什么一个人学了很多东西，却很难改变真实行动。

课程原文：
很多人以为学习的障碍是信息不够，但真正的障碍往往是场景不够。
如果一个知识点没有进入你的任务、关系、选择和反馈里，它就只是一个
漂亮的说法。知识不是因为被理解而改变人，而是因为被反复调用、检验
和修正，才慢慢变成新的判断力。

所以学习一堂课，不能只问“老师说了什么”，还要问“这件事在我的生活
里对应哪个真实问题”。如果找不到这个问题，课程就会停留在摘要层。
```

Expected first assistant response:

```text
这堂课材料已经够了。先确认学习方式：这堂课要不要接入你已有笔记？选一个就行：不接入 / 扫描指定文件夹 / 接入 Obsidian 库。
```

Result: pass.

The first response is short, asks only one setup question, and does not start a long configuration interview.

## No-Knowledge Mode Simulation

User answer:

```text
不接入，先只基于这堂课聊。
```

Expected behavior:

1. Resolve `knowledge.mode = none`
2. Call `three-level-notes` with the full lesson text
3. Verify Part 1 / Part 2 / Part 3 are present
4. Extract 3-5 internal anchors instead of scanning files
5. Share a real opening reaction and ask one focused question

Expected Phase 2 opening:

```md
这次不接入旧笔记。我先抓 3 个课程内部锚点：

1. **信息不等于改变**：课程把“知道”和“行动变化”切开了。
2. **场景是知识的入口**：知识要进入任务、关系、选择和反馈，才会产生力量。
3. **真实问题决定学习深度**：找不到自己的问题，课程就容易停留在摘要层。

我的真实反应是：这堂课很准，但它还没讲透一个细节：很多人不是不想把知识放进真实场景，而是不知道自己正在被哪个真实问题困住。所以课程提醒得对，但真正难的是识别“我现在到底在用这个知识解决什么”。

我想先问你一个问题：这堂课里哪句话最像是在说你最近的某个具体处境？
```

Result: pass with one test limitation.

Limitation: this local test did not invoke a real nested Skill runtime. It verified that the public package clearly requires `three-level-notes` and that the dependency is present locally.
