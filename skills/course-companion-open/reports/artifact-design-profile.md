# Artifact Design Profile

## Artifact Family

Markdown learning artifacts:

- final study note
- optional knowledge card
- config file

## Reading Experience

The final note should feel like a clear study record, not a transcript. It should preserve the course structure, show knowledge connections, and capture the highest-value dialogue moments.

## Structure Rules

- Use headings that explain the content, not generic labels.
- Use bullets for short claims and callouts for quoted dialogue.
- Use tables only when comparing modes or options.
- Keep related notes short: title/path plus one reason.
- Keep knowledge cards standalone and reusable.

## Quality Checks

- The note can be understood without rereading the whole chat.
- The user's view, assistant's view, and course author's view are distinguishable.
- The knowledge card has one clear concept, not a bundle of loosely related ideas.
- No local private path appears unless supplied by the user in that run.
