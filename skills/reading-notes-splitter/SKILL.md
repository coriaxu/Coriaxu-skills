# 读书笔记拆分器

将混乱的 HTML 读书笔记转换为清晰的 Markdown 卡片格式，每张卡片都包含：来源、标题、个人想法和原书摘要。

## 使用场景

- 导出的 flomo 读书笔记需要整理
- HTML 格式的笔记需要转换为 Markdown
- 笔记混在一起，需要拆分成独立卡片
- 需要区分"我的想法"和"原书摘要"

## 工作流程

### 第 1 步：定位和读取 HTML 文件

首先列出当前目录的文件，找到 HTML 格式的读书笔记文件：

```bash
ls -la
```

然后读取 HTML 文件，了解笔记的结构和数量。

### 第 2 步：分析笔记结构

HTML 笔记通常包含：
- `.memo` 类：每条笔记的容器
- `.time` 类：笔记创建时间
- `.content` 类：笔记内容主体
- 标签（如 `#得到/电子书/书名`）
- 分隔符（如 `----` 或 `————`）

**识别规则**：
- **标签行**：以 `#` 开头的标签，提取书名和来源
- **我的想法**：
  - 以 🤔、💡、📝、🤖 等 emoji 开头的段落
  - 加粗标记的 `<strong><mark>idea xxx</mark></strong>` 段落
  - 分隔符之前的所有内容
- **标题**：
  - `<strong><mark>idea xxx</mark></strong>` 中的文字
  - 或我的想法部分的第一句话
- **原书摘要**：分隔符（`----` 或 `————`）之后的内容
- **来源链接**：通常在最后一行 `来源：https://...`

### 第 3 步：创建任务列表

使用 TodoWrite 工具创建任务列表来追踪进度：

```
1. 分析 HTML 文件结构，理解笔记格式
2. 解析每条笔记，提取标题、想法和原书摘要
3. 创建 Markdown 文档，按卡片格式整理
4. 检查确保所有笔记都已转换，无遗漏
```

### 第 4 步：提取和整理笔记

对每条 `.memo` 笔记：

1. 提取时间戳（`.time` 的内容）
2. 提取标签，识别书名
3. 找到分隔符位置
4. 分隔符之前 = 我的想法（去掉标签行）
5. 分隔符之后、来源链接之前 = 原书摘要
6. 提取标题（从想法的第一段或 `<mark>` 标签）

### 第 5 步：创建 Markdown 文档

创建以书名命名的 `.md` 文件，结构如下：

```markdown
# 《书名》 - 读书笔记

> 作者读书笔记整理
> 笔记来源：flomo 浮墨笔记
> 导出日期：YYYY-MM-DD
> 共 X 条笔记

---

## 卡片 1

**【来源】** 《书名》- 来源平台

**【标题】** 卡片标题

**【我的想法/Idea】**

我的思考和感想内容...

**【原书摘要】**

原书的文字内容...

**【记录时间】** YYYY-MM-DD HH:MM:SS

---

## 卡片 2

...
```

### 第 6 步：验证完整性

检查：
- HTML 中有 N 条 `.memo`，Markdown 中应该有 N 张卡片
- 每张卡片都包含所有必需字段
- 时间排序正确（通常按时间正序）
- 无内容遗漏或格式错误

## 输出规范

### 文件命名
- 使用书名作为文件名
- 格式：`书名.md`（不带书名号）
- 如：`新生育时代.md`

### 卡片格式
每张卡片必须包含：
- ✅ 【来源】书名和平台
- ✅ 【标题】简洁的主题概括
- ✅ 【我的想法/Idea】完整的个人思考
- ✅ 【原书摘要】原文引用
- ✅ 【记录时间】完整时间戳

### 排序规则
- 默认按时间正序（从早到晚）
- 便于看到思考的演进过程

### 分隔符
- 使用 `---` 清晰分隔每张卡片
- 每张卡片之间空一行

## 常见问题处理

### 识别标题
如果笔记有 `<strong><mark>idea XXX</mark></strong>`，用这个作为标题；
否则，从"我的想法"的第一段提取核心主题。

### 处理 emoji
保留所有 emoji（🤔💡📚 等），它们是想法的重要标记。

### 处理 AI 助理回答
如果笔记中包含 "🤖 小助理回答如下" 或类似内容，全部保留在【我的想法】部分。

### 处理长篇想法
如果想法部分很长（如包含多个段落、列表等），全部保留，不要截断。

### 处理分隔符变体
常见分隔符：
- `--------------------`（20 个短横线）
- `————`（4 个长破折号）
- `---`（3 个短横线）

都应识别为分隔符。

### 处理来源链接
- 提取书名和平台信息
- 完整的 URL 可保留在文档末尾的统计部分

## 示例

### 输入：HTML 笔记片段
```html
<div class="memo">
  <div class="time">2025-10-08 11:06:02</div>
  <div class="content">
    <p>#得到/电子书/新生育时代</p>
    <p>🤔 让 GPT5 thinking 做了个预测，15 年之后再看看👀</p>
    <p>--------------------</p>
    <p>中国人口与发展研究中心的研究显示...</p>
    <p>来源：https://dedao.cn/ebook/...</p>
  </div>
</div>
```

### 输出：Markdown 卡片
```markdown
## 卡片 1

**【来源】** 《新生育时代》- 得到电子书

**【标题】** 女性终身无孩率预测

**【我的想法/Idea】**

🤔 让 GPT5 thinking 做了个预测，15 年之后再看看👀

**【原书摘要】**

中国人口与发展研究中心的研究显示...

**【记录时间】** 2025-10-08 11:06:02
```

## 使用提示

这个 skill 会自动：
- 识别 HTML 中的笔记结构
- 拆分每条笔记
- 创建清晰的卡片格式
- 保留所有原始内容
- 按时间排序

你只需要：
1. 确保 HTML 文件在当前目录
2. 提到"读书笔记拆分"或"flomo 转 MD"等关键词
3. 让我完成剩余工作

转换完成后，你会得到：
- 一个以书名命名的 .md 文件
- 清晰分隔的卡片
- 完整的统计信息
- 无遗漏的内容

## 技术说明

### 使用的工具
- `Bash`: 列出文件
- `Read`: 读取 HTML 文件
- `Write`: 创建 Markdown 文件
- `TodoWrite`: 追踪任务进度

### 处理流程
1. 使用 TodoWrite 创建任务列表
2. 使用 Bash 列出目录文件
3. 使用 Read 读取 HTML 内容
4. 解析并提取笔记信息
5. 使用 Write 创建 Markdown 文档
6. 标记任务完成

### 质量保证
- 验证笔记数量匹配
- 确保所有字段完整
- 检查格式一致性
- 确认无内容遗漏
